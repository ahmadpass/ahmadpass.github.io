GitHub: SSH Connect

Easily connect your GitHub account into your PC using SSH

Posted by Ahmad R.A on October 2, 2024

For Linux users, accessing your GitHub account using SSH (Secure Shell) requires a bit of work, especially if you are using a command line. It will not work by just providing the username and password. Prior, we needed to provide our SSH key (public key) to GitHub, so the connection could be maintained.

Before we begin, we should check whether Git is already installed on our terminal. 

nate@PC-1701:~$ git --version
git version 2.43.0

Then, let's try to push some content into our GitHub account using a command line. It should look like this. 

nate@PC-1701: ~/Documents/Projects/WebPage/ahmadpass.github.io$ git push -u origin main
Username for 'https://github.com': ahmadpass
Password for 'https://ahmadpass@github.com':
remote: Permission to ahmadpass/ahmadpass.github.io.git denied to ahmadpass.
fatal: unable to access 'https://github.com/ahmadpass/ahmadpass.github.io/': The requested URL returned error: 403
Connection to our GitHub account denied

When we try to connect to our GitHub account, we will be prompted to enter our username and password. Although now the password has already been replaced by a token for security measures (See Personal Access Tokens for more info). If you haven't already set up a token, follow from the Step 1. If you are done with the token, you can skip to Step 2â€‹.

Step 1: Set Up GitHub token

To do this, navigate to your GitHub account > Click on your Profile > Settings > On the left side, below "Archives", click on Developer settings.

Click on "Generate new token" to create ones

Choose Generate new token > new personal access token (classic). Next we will be prompted to enter our password. Then create your new token by filling out the detail as depicted below.

Fill out the details as described

When you are done, click on Generate token button at the bottom. Then you will see this.

Copy the generated token

Copy the generated token, and stored it somewhere safe as it will be dissapear once you leave the page. We will need this token later when we are going to connect again using GitHub API from the command line.

Step 2: Create the SSH keys

First, make sure the ssh-agent is running by using this command. 

nate@PC-1701:~$ eval "$(ssh-agent -s)"
Agent pid 23443

Now we are going to create a pair of SSH keys, using the command below. 

ssh-keygen -t rsa

The "t" switch above specify the type of encryption key, which is RSA. After running the command, we will be prompted to enter the file name and passphrase. You can freely named the key file. In this case, I will go with "id_rsa". Then you can skip or fill the passphrase. This is used to encrypt the SSH key, to make your key more secure.

The result of the command above

By default, the generated key should be stored inside "/home/<User>/.ssh" folder. We can see now there is a pair of keys. The id_rsa is the private key, we should never share this with anyone. While the id_rsa.pub is the one we are going to share with our GitHub account.

Now we should add the private key identities to the authentication agent (ssh-agent), using the command below. 

ssh-add <path to private key>
Add the private key to ssh-agent

This way, we will be able to authenticate the connection from GitHub API. To see the list of private key on the ssh-agent we can use the command: 

ssh-add -l
The private key on ssh-agent

Step 3: Associate the SSH key to the GitHub account

Now go back to the location where we put the public key before. In this case, the key is stored in "/home/<User>/.ssh". See the content of the id_rsa.pub (don't reverse it with the private ones) using "cat" or "vim" command, then copy it. Now go to your GitHub account > Click on your Profile > Settings > On the left side under "Access" you will find SSH and GPG keys, click on it > On "SSH Keys" sub tab, click on New SSH key.

Add new SSH key on GitHub account

Now you can paste the content of your public key inside the Key text box. Then click on Add SSH key after you are done with the details.

Now your public key has been associated with the GitHub account

Compare the SSH key with the output from ssh-add -l command before to make sure it's the correct key.

Connect!

Now let's we try to push our repository again using the command git push. Enter the username and password if prompted. Remember, use the generated token we created previously in place of your password.

Now the connection is not being denied

You should now be able to connect to your GitHub account using SSH. Doesn't seem too complicated, doesn't it?

Extra

If you are tryng to connect to your GitHub from Visual Studio Code, and faces this issue:

Issue while tryng to connect using vs code

Check the configured user.name and user.email using this command. 

git config --list --show-origin

If the value is different than you expected, or even empty, configure the correct value using this command. 

git config --global user.name "John Doe"
git config --global user.email johndoe@example.com
